Module 2: Structuring Content in a Web Document

Use semantic HTML5 elements to give meaning to your content — better for users, search engines, and accessibility.

🏛️

Semantic HTML5 Layout Elements

Tags that describe their purpose, not just their appearance.

Why Semantics Matter

A <div> says nothing about what it contains. A <nav> tells everyone — browsers, screen readers, search bots — that this is navigation. Semantic HTML improves:

  • Accessibility — screen readers announce landmark regions.
  • SEO — search engines understand your page structure.
  • Maintainability — other developers instantly understand the layout.

Full Page Layout Example

<body> <header> <!-- site logo, banner --> <nav>…</nav> <!-- main navigation --> </header> <main> <!-- unique page content (only ONE per page) --> <section> <!-- thematic grouping --> <article> <!-- self-contained content (blog post, card) --> <h2>Post Title</h2> <p>Content…</p> </article> </section> <aside>…</aside> <!-- sidebar, related links --> </main> <footer>…</footer> <!-- copyright, contact info --> </body>

🏗️ HTML5 Landmark Elements

<header> Site/section header (logo, banner) <nav> Navigation links <main> Primary content (unique, one per page) <section> Thematic grouping with a heading <article> Self-contained syndicate-able content <aside> Secondary content / sidebar <footer> Bottom info: copyright, links <address> Contact info for nearest article/body

📝 Text Structure Elements

<p>Paragraph — the basic text block</p> <blockquote> Longer quotation from another source. </blockquote> <figure> <img src="chart.png" alt="Sales chart"> <figcaption>Q4 2024 Sales</figcaption> </figure> <details> <summary>Click to expand</summary> Hidden content revealed on click. </details>

📊 Heading Hierarchy

<!-- ✅ CORRECT — one h1, logical order --> <h1>Page Title</h1> <h2>Chapter 1</h2> <h3>Sub-section</h3> <h2>Chapter 2</h2> <!-- ❌ WRONG — skipping levels, multiple h1s --> <h1>Title</h1> <h1>Another Title</h1> <!-- Only one h1! --> <h4>Skipped h2 & h3!</h4>
💡

Each page should have exactly one <h1>. Never skip heading levels (h1 → h3) as it breaks accessibility.
