Chrome DevTools & Lighthouse

Your built-in performance lab — no installs required.

Opening DevTools

/* Open Chrome DevTools */ Windows / Linux : F12 or Ctrl + Shift + I macOS : Cmd + Option + I Right-click any element → "Inspect" /* Key panels for performance */ Elements — inspect & live-edit HTML/CSS Console — JS errors, logs, quick tests Network — see every file loaded, size, timing Performance— record and profile page runtime Lighthouse — automated audit: perf, a11y, SEO, PWA

Running a Lighthouse Audit

/* DevTools → Lighthouse tab */ 1. Open DevTools (F12) 2. Click "Lighthouse" tab 3. Choose: Mobile or Desktop 4. Select categories: Performance ✓ Accessibility ✓ SEO ✓ 5. Click "Analyze page load" 6. Review scores (0-100) and fix "Opportunities" /* Or run from the command line */ npx lighthouse https://example.com --view
💡

Run Lighthouse in Incognito mode for accurate results — browser extensions inflate load times and can skew scores.

⚡ Core Web Vitals (CWV)

/* Google's user-experience metrics — affect search ranking */ LCP — Largest Contentful Paint When the biggest visible element loads. ✅ Good: < 2.5 s ⚠️ Needs work: 2.5-4 s ❌ Poor: > 4 s Fix: optimize hero images, use fetchpriority="high" INP — Interaction to Next Paint (replaced FID in 2024) Responsiveness to clicks, taps, key presses. ✅ Good: < 200 ms ⚠️ Needs work: 200-500 ms ❌ Poor: > 500 ms Fix: reduce long JS tasks, avoid layout thrashing CLS — Cumulative Layout Shift How much page content shifts unexpectedly. ✅ Good: < 0.1 ⚠️ Needs work: 0.1-0.25 ❌ Poor: > 0.25 Fix: always set width & height on <img> and <video>
💡

CLS is most often caused by images without dimensions. Always add width and height attributes to every <img> — the browser reserves the space before the image loads.

🖼️ Image Optimization

<!-- 1. Choose the right format --> Photos → AVIF > WebP > JPEG Transparency → WebP > PNG Icons / logos → SVG (infinitely scalable, tiny file) Animation → WebM video (much smaller than GIF!) <!-- 2. Serve the right size --> <img src="hero-800.webp" srcset="hero-400.webp 400w, hero-800.webp 800w, hero-1200.webp 1200w" sizes="(max-width:600px) 100vw, 800px" width="800" height="450" loading="lazy" decoding="async" alt="Hero image" > <!-- 3. Prioritize the hero / LCP image --> <img src="hero.webp" fetchpriority="high" loading="eager" alt="..."> /* 4. Target file sizes */ Above-the-fold hero → < 200 KB Thumbnails / cards → < 50 KB Icons (SVG) → < 5 KB

⚙️ CSS & JavaScript Performance

/* CSS — render-blocking by default, so keep it lean */ /* ✅ Load non-critical CSS asynchronously */ <link rel="preload" href="styles.css" as="style" onload="this.rel='stylesheet'"> /* ✅ Inline critical CSS (above-the-fold styles) in <head> */ <style> body { margin:0; font-family:sans-serif; } header { background:#1e293b; color:#fff; } </style> /* ✅ will-change: hint the browser to promote element to GPU */ .animated-card { will-change: transform; } /* ❌ Avoid triggering layout thrash */ /* Don't read offsetHeight then set height in a loop — batch reads/writes */ /* JavaScript */ <!-- ✅ defer: load after HTML parsed, execute after DOM ready --> <script src="app.js" defer></script> <!-- ✅ async: download in parallel, execute immediately --> <script src="analytics.js" async></script>

📦 Caching & Font Loading

/* HTTP Cache headers — set on the server */ Cache-Control: max-age=31536000, immutable ← versioned assets (JS, CSS) Cache-Control: no-cache ← HTML (always revalidate) ETag: "abc123" ← fingerprint for validation /* Font loading best practices */ <!-- 1. Preconnect to font CDN early --> <link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> /* 2. Use font-display: swap — show text immediately in fallback */ @font-face { font-family: 'Inter'; src: url('inter.woff2') format('woff2'); font-display: swap; /* prevents invisible text (FOIT) */ } /* 3. Subset fonts — only include needed characters */ Google Fonts does this automatically via ?subset=latin Self-hosted: use pyftsubset or glyphhanger CLI tools
💡

font-display: swap prevents the "Flash of Invisible Text" (FOIT) — instead of hiding text while the font loads, the browser shows it in a system fallback font and swaps when the custom font is ready.

🌐 Reading the Network Tab

/* DevTools → Network tab: what to look for */ Waterfall — each row is a file; long rows = slow resource Size — "(from cache)" = good! means browser cached it Time — TTFB (Time To First Byte) > 600ms = slow server Initiator — what triggered the request (HTML, JS, CSS) Status — 200 ✅ 301/302 redirect ⚠️ 404 not found ❌ /* Quick wins you can spot in Network tab */ Large unoptimized images → convert to WebP, add srcset Missing cache headers → add Cache-Control to server Render-blocking CSS/JS → add defer / preload Too many HTTP requests → bundle files or use HTTP/2 Slow TTFB → upgrade hosting or use a CDN

🏆 Performance Quick-Win Checklist

/* HTML */ ✅ Add width & height to every <img> → fixes CLS ✅ Add loading="lazy" to below-fold images → faster LCP ✅ Add fetchpriority="high" to hero image → faster LCP ✅ Add defer to <script> tags → unblocks parsing ✅ Minify HTML (remove comments & whitespace) → smaller file /* CSS */ ✅ Inline critical (above-fold) CSS in <head> → no render block ✅ Remove unused CSS (PurgeCSS, DevTools Coverage tab) → smaller file ✅ Use will-change: transform on animated elements → GPU layer ✅ Prefer CSS animations over JS for motion → smoother 60fps /* Images */ ✅ Convert to WebP / AVIF → 30-80% smaller ✅ Use srcset + sizes for responsive delivery → right size per screen ✅ Compress with Squoosh, TinyPNG, or Sharp → less bandwidth /* Fonts */ ✅ Preconnect to font CDN → faster DNS ✅ Use font-display: swap → no FOIT ✅ Limit font weights (e.g. 400 & 700 only) → fewer requests /* Server / Hosting */ ✅ Enable Gzip / Brotli compression → 60-80% smaller transfers ✅ Set Cache-Control headers → repeat visits instant ✅ Use a CDN (Cloudflare, Vercel, Netlify) → global fast delivery
