Module 9: Implementing Responsive Design

Build layouts that look great on every screen — from a 320px phone to a 4K monitor. Master media queries, fluid grids, and the mobile-first approach.

@media mobile-first viewport fluid layout breakpoints clamp()
📱

Why Responsive Design?

Over 60% of web traffic comes from mobile devices.

The Viewport Meta Tag — Required First Step

Without this tag, mobile browsers zoom out to fit the desktop layout — your responsive CSS will be ignored.

<!-- REQUIRED in every <head> --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- width=device-width → use actual screen width initial-scale=1.0 → no zoom on load -->

Common Breakpoints (reference, not rules)

/* Tailwind-inspired breakpoints — widely used */ sm: @media (min-width: 640px) /* large phones */ md: @media (min-width: 768px) /* tablets */ lg: @media (min-width: 1024px) /* laptops */ xl: @media (min-width: 1280px) /* desktops */ 2xl: @media (min-width: 1536px) /* wide screens */
💡

Let content dictate breakpoints. Add a breakpoint when the layout breaks, not at every standard device size.

📱 The Mobile-First Approach

Write base CSS for mobile, then use min-width queries to enhance for larger screens.

/* ✅ MOBILE-FIRST — start small, scale up */ .grid { display: grid; grid-template-columns: 1fr; /* 1 col: mobile */ gap: 16px; } @media (min-width: 640px) { .grid { grid-template-columns: repeat(2, 1fr); } /* tablet */ } @media (min-width: 1024px) { .grid { grid-template-columns: repeat(3, 1fr); } /* desktop */ } /* ❌ DESKTOP-FIRST — harder to maintain */ .grid { grid-template-columns: repeat(3, 1fr); } @media (max-width: 1024px) { .grid { grid-template-columns: repeat(2, 1fr); } } @media (max-width: 640px) { .grid { grid-template-columns: 1fr; } }

📐 Media Query Syntax

/* Basic syntax */ @media media-type (feature: value) { /* CSS rules */ } /* Screen only (vs print) */ @media screen and (min-width: 768px) { ... } /* OR condition */ @media (max-width: 480px), (orientation: portrait) { ... } /* Combining min AND max */ @media (min-width: 640px) and (max-width: 1023px) { ... } /* Dark mode preference */ @media (prefers-color-scheme: dark) { body { background: #0f172a; color: #f8fafc; } } /* Reduced motion — respect user preferences */ @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; } } /* Print styles */ @media print { nav, footer, .ads { display: none; } }

💧 Fluid Typography & Layout

/* clamp(min, preferred, max) — fluid without media queries */ h1 { font-size: clamp(1.8rem, 5vw, 3.5rem); } p { font-size: clamp(1rem, 1.5vw, 1.25rem); } /* Fluid padding */ .section { padding: clamp(24px, 5vw, 80px) clamp(16px, 4vw, 48px); } /* Auto-responsive grid — no media queries needed! */ .auto-grid { display: grid; grid-template-columns: repeat(auto-fill, minmax(240px, 1fr)); gap: clamp(12px, 2vw, 24px); } /* Container with max-width */ .container { width: min(100% - 48px, 1200px); /* never wider than 1200px */ margin-inline: auto; /* centered */ } /* Responsive images */ img { max-width: 100%; height: auto; }

🧭 Responsive Navigation Pattern

<!-- HTML --> <nav class="navbar"> <a href="/" class="logo">MyBrand</a> <button class="hamburger" aria-label="Toggle menu" aria-expanded="false" onclick="toggleNav(this)">☰</button> <ul class="nav-links"> <li><a href="/">Home</a></li> <li><a href="/about">About</a></li> <li><a href="/contact">Contact</a></li> </ul> </nav> /* CSS */ .hamburger { display: block; } /* visible on mobile */ .nav-links { display: none; } /* hidden on mobile */ .nav-links.open { display: flex; flex-direction: column; } @media (min-width: 768px) { .hamburger { display: none; } /* hide on tablet+ */ .nav-links { display: flex; gap: 24px; } }

Testing Responsive Layouts

Use DevTools to simulate any device without a physical phone.

Browser DevTools — Device Emulator

  • Press F12 (or right-click → Inspect) to open DevTools.
  • Click the Toggle Device Toolbar icon (📱) or press Ctrl+Shift+M.
  • Select a preset device (iPhone, Pixel, iPad) or enter a custom width.
  • Drag the viewport edge to test different widths and find where your layout breaks.
