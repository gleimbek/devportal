CSS Transitions

Smooth property changes on state changes like :hover.

transition — The Basics

/* transition: property duration timing-function delay */ .btn { background: #1d4ed8; color: #fff; padding: 12px 28px; border-radius: 8px; transition: background 0.3s ease, transform 0.2s ease; } .btn:hover { background: #1e40af; transform: translateY(-2px); } /* Transition all properties (use carefully) */ .card { transition: all 0.25s ease-in-out; } /* Timing functions */ /* ease | linear | ease-in | ease-out | ease-in-out | cubic-bezier() */ .smooth { transition-timing-function: cubic-bezier(0.34, 1.56, 0.64, 1); }

CSS Transforms

/* 2D Transforms */ .move { transform: translate(50px, 20px); } .rotate { transform: rotate(45deg); } .scale { transform: scale(1.5); } .skew { transform: skew(10deg, 5deg); } /* Combine transforms */ .combo { transform: translateY(-4px) scale(1.02); } /* 3D Transforms */ .flip { transform: rotateY(180deg); } .perspective-card { transform-style: preserve-3d; perspective: 1000px; }

🎬 @keyframes Animation

/* Define the animation steps */ @keyframes fadeInUp { from { opacity: 0; transform: translateY(30px); } to { opacity: 1; transform: translateY(0); } } /* Multi-step animation */ @keyframes bounce { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-30px); } } /* Apply the animation */ .hero-text { animation: fadeInUp 0.6s ease-out both; } .icon { animation: bounce 1s ease infinite; }

⚙️ Animation Properties

.element { animation-name: fadeInUp; animation-duration: 0.6s; animation-timing-function: ease-out; animation-delay: 0.2s; animation-iteration-count: 1; /* or infinite */ animation-direction: normal; /* reverse | alternate */ animation-fill-mode: both; /* forwards | backwards | both */ animation-play-state: running; /* paused on hover */ /* Shorthand */ animation: fadeInUp 0.6s ease-out 0.2s 1 normal both; } /* Performance tip: animate only transform & opacity */ .gpu { will-change: transform, opacity; }

♿ Respecting Motion Preferences

/* Some users experience motion sickness. Disable animations for them! */ @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } } /* Or opt-in approach — only animate if user is OK with it */ @media (prefers-reduced-motion: no-preference) { .animate-on-scroll { animation: fadeInUp 0.5s ease both; } }
💡

Always include a prefers-reduced-motion media query. It's a small addition that makes a big difference for users with vestibular disorders.
