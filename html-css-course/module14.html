Open Graph Protocol

Control how your page appears when shared on social media.

Open Graph Meta Tags

When someone shares your URL on Facebook, LinkedIn, WhatsApp, or Slack, these tags control the preview card that appears.

<head> <!-- Essential Open Graph tags --> <meta property="og:title" content="My Awesome Article"> <meta property="og:description" content="A concise summary under 200 chars."> <meta property="og:image" content="https://example.com/social-preview.jpg"> <meta property="og:url" content="https://example.com/article"> <meta property="og:type" content="article"> <meta property="og:site_name" content="My Website"> <meta property="og:locale" content="en_US"> <!-- og:image specs: 1200×630px recommended, <5MB --> <meta property="og:image:width" content="1200"> <meta property="og:image:height" content="630"> <meta property="og:image:alt" content="Article cover image"> </head>
💡

The og:image is the most important tag — it's the first thing people see when your link is shared. Design a dedicated 1200×630px image for each key page.

🐦 Twitter / X Card Tags

<!-- Twitter uses its own meta tags (also reads og: as fallback) --> <meta name="twitter:card" content="summary_large_image"> <meta name="twitter:site" content="@YourHandle"> <meta name="twitter:creator" content="@AuthorHandle"> <meta name="twitter:title" content="My Awesome Article"> <meta name="twitter:description" content="Summary of the article."> <meta name="twitter:image" content="https://example.com/card.jpg"> <meta name="twitter:image:alt" content="Card image description"> <!-- Card types --> /* summary — small image + text summary_large_image — full-width image (most common) app — mobile app cards player — embedded media player */

🔗 Custom Share Buttons (no SDK needed)

<!-- Build share links manually — no tracking scripts! --> <!-- Twitter / X --> <a href="https://twitter.com/intent/tweet ?url=https://example.com/article &text=Check+this+out! &via=YourHandle" target="_blank" rel="noopener noreferrer"> Share on X </a> <!-- Facebook --> <a href="https://www.facebook.com/sharer/sharer.php ?u=https://example.com/article" target="_blank" rel="noopener noreferrer"> Share on Facebook </a> <!-- LinkedIn --> <a href="https://www.linkedin.com/shareArticle ?url=https://example.com/article &title=My+Article" target="_blank" rel="noopener noreferrer"> Share on LinkedIn </a> <!-- Native Web Share API (mobile / modern browsers) --> <button onclick="navigator.share({title:'Article',url:location.href})"> 📤 Share </button>

📌 Embedding Social Posts

<!-- Instagram: use blockquote embed from Instagram --> <blockquote class="instagram-media" data-instgrm-permalink="https://www.instagram.com/p/POST_ID/"> </blockquote> <script async src="//www.instagram.com/embed.js"></script> <!-- X / Twitter post embed --> <blockquote class="twitter-tweet"> <a href="https://twitter.com/user/status/TWEET_ID"></a> </blockquote> <script async src="https://platform.twitter.com/widgets.js"></script> <!-- Alternative: screenshot or static quote card (no JS overhead, GDPR friendly) --> <figure> <img src="tweet-screenshot.png" alt="Tweet by @user: Full tweet text here"> <figcaption>Via <a href="https://twitter.com/...">@user on X</a></figcaption> </figure>

🔒 Privacy & Performance Tips

/* Social SDK scripts load third-party JS on your page. This can: - Slow down page load (render-blocking) - Track visitors without their consent (GDPR/CCPA) - Inject additional cookies Better approaches: */ 1. Use manual share links (no SDK required) → Just URL-based links, zero JS overhead 2. Lazy-load social embeds → Load the SDK only when user clicks "Show Tweet" 3. Use screenshots for social proof → Static image, GDPR-safe, fast 4. Add loading="lazy" to iframes → Defers off-screen embed loading
⚠️

Social media SDKs (Like buttons, Follow buttons) load tracking scripts even if the user never interacts with them. Under GDPR/CCPA, you may need user consent before loading them.
